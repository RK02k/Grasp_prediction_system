Key observations:

1. PINCH GRASP IS CATASTROPHICALLY MISCALIBRATED — predicted 43 times
   (most frequent), correct only once (2%). The model defaults to "pinch"
   as a generic fallback for any small or ambiguous object.

2. BUTTON-PRESS GRASP IS NEVER CORRECT (0/12). The model applies it to
   objects that don't warrant it.

3. CYLINDRICAL GRASP IS MOST RELIABLE IN VOLUME — 43% accuracy across 42
   predictions. Likely because cylindrical objects (bottles, mugs) are
   visually unambiguous.

4. LATERAL PINCH IS MOST RELIABLE BY RATE — 82% accuracy. When the model
   commits to it, it is usually right.

================================================================
5. EXACT ACCURACY = SEMANTIC ACCURACY (BROKEN SEMANTIC FALLBACK)
================================================================

Across all 30 scenes, exact and semantic accuracy are identical or differ
by only ±1%. The semantic similarity layer is adding zero value.

The model's grasp prediction is either exactly right or in a completely
different semantic cluster. There is no "close but not exact" case being
rescued.

Possible causes:
  (a) The semantic taxonomy groups are too coarse — most wrong predictions
      fall outside all nearby groups.
  (b) The semantic scoring is matching against the wrong GT entry because
      the object was misidentified in Stage 2 (most likely).
  (c) The semantic mapping is not implemented with a sufficiently loose
      threshold.

  2        Pinch grasp over-prediction              Destroys grasp accuracy (2% acc)

  6        Semantic fallback non-functional         Exact == Semantic everywhere


================================================================
9. RECOMMENDED NEXT STEPS
================================================================

1. FIX SM3 OBJECT MATCHING
   Add fuzzy/synonym normalisation so "plate" -> "dish",
   "head & shoulders Supreme" -> "head_shoulders_supreme", etc.
   This alone would recover many missed GT matches and improve recall
   significantly.

2. SUPPRESS PINCH GRASP BIAS
   Either add a prompt constraint listing valid grasp types per object
   shape, or add a post-hoc re-scoring step penalising over-use of
   "Pinch grasp". Target: reduce from 43 predictions to <15.

3. DEBUG SM1 ON ZERO-RECALL SCENES
   scene_0013 lost all 8 masks before VLM. Investigate whether the
   SM1 object/background classifier threshold is too aggressive. Consider
   logging the SM1 confidence scores per mask.

4. RE-IMPLEMENT SEMANTIC GRASP SIMILARITY
   The exact=semantic parity indicates it is not functioning. Implement
   proper grouping (e.g., power grasp family: cylindrical + spherical;
   precision family: pinch + lateral pinch) with explicit fallback logic.

5. IMPROVE OOD OBJECT RECOGNITION
   Options:
     (a) Add few-shot examples in the SM3 prompt for known OOD objects
     (b) Replace 3B model with Qwen2.5-VL-7B for SM3 only
     (c) Add a post-processing synonym map for known dataset objects

6. ADDRESS DUPLICATE DETECTIONS
   Banana was identified 4x in scene_0000 (only 1 exists). Add NMS
   (non-maximum suppression) on object names per scene after SM3.
